# Pandora PvP 规则待定项

> 本文档是**待用户拍板的 backlog**,所有 ❓ 项需要决策后才能写代码。
> 先按 §6 默认配置写代码,后期策划再调,改 1 行 config 即可,不影响架构。

## 1. 大厅 PvP 基本规则

### 1.1 死亡惩罚 ❓

**待定方案**:
- A. **轻惩罚**(推荐 MOBA-Lite):重生 5s,无任何掉落,纯娱乐 PvP
- B. **中惩罚**:掉一些金币,无装备掉落
- C. **重惩罚**(类 Albion):掉装备 + 经验
- D. **混合**:特定区域 A,黑色区域 C

**影响**:
- A → 适合休闲玩家
- B/C → 玩家会"逃离 PvP"
- D → 设计复杂但平衡好

**关联系统**:复活点 / 拾取系统 / 库存系统

### 1.2 新人保护 ❓

**待定方案**:
- A. 等级 < 10 的玩家,**无法被攻击也无法攻击**(免战)
- B. 等级 < 10 的玩家,只能在"新手区" PvP
- C. 不保护,丛林法则

**推荐 A**(用户友好,但要做"新手区→主城"过渡引导)

### 1.3 击杀奖励 ❓

**待定方案**:
- 金币:固定 X / 受害者持有的 Y%
- 经验:固定 X / 段位差加成
- **连杀奖励**:连杀 3/5/10 解锁额外效果
- **赏金机制**:被多人击杀过的玩家头顶有"赏金" buff

### 1.4 PvP 安全区 ❓

**待定方案**:
- A. **NPC 周围 30m 半径**(出生点、商店、传送点)→ 安全区无法主动开战,被攻击则脱离
- B. **整个城镇有"主城安全区"**,只有"野外"才 PvP
- C. **无安全区**(强 PvP)

**推荐 A**(让 NPC 服务能正常工作)

### 1.5 阵营 / 仇敌 ❓

- 是否有阵营(部落 / 联盟)?
- 同阵营是否能 PvP?
- 还是无阵营,全员可互打?

**推荐**:无阵营,全员可互打,但有"组队保护"——同队不能互伤

### 1.6 死亡通知 / 标记

- 击杀消息是否广播?
- 击杀者头顶是否有标记 / 仇敌系统?

## 2. 战斗 DS PvP 规则(MOBA 标准)

### 2.1 模式

- 5v5 经典模式(20~30 分钟)
- 是否有 1v1 / 3v3 / 大乱斗?(Phase 2)

### 2.2 投降

- 几人同意投降?(LoL 4 / DOTA2 全员)
- 几分钟开始能投降?(LoL 15min / DOTA2 任意)

### 2.3 离线惩罚

- 掉线 / AFK 多久判负?
- 段位下降幅度?
- 短时间多次掉线 → 排队限制?

### 2.4 段位系统 ❓

**段位规划**:
- 青铜 → 白银 → 黄金 → 铂金 → 钻石 → 大师 → 王者
- 每段位是否分小级(青铜 V → I)?
- MMR 隐藏分 vs 段位明分关系?
- 赛季机制:每 3 个月重置?段位回退多少?

### 2.5 MMR 算法

- ELO / TrueSkill / Glicko-2?
  - **推荐 Glicko-2**(考虑波动率,适合 MOBA)
- 队伍 MMR = 队员平均还是加权?
- 与对手段位差影响输赢分?

### 2.6 禁选英雄

- 是否有 ban 阶段?
- 几个 ban 位?(LoL 10 / DOTA2 14)
- ban 顺序?

## 3. 反作弊规则

### 3.1 移动校验

- 服务端记录上次位置 + 时间戳
- 当前位置距离 / 时间 > maxSpeed × 1.2 → 标记 + kick

### 3.2 技能 CD 校验

- 服务端权威 CD
- 客户端宣称的 ability activation 时间戳必须 ≥ last + cd
- 不满足 → 拒绝

### 3.3 伤害校验

- 服务端独立计算伤害(不信客户端)
- 客户端只发"我对 X 用 Y 技能",其余服务端算

### 3.4 异常行为日志

- 写 `pandora.security.audit` topic
- 每日批处理风控

### 3.5 封禁规则 ❓

- 自动封禁阈值?(误封风险)
- 申诉流程?
- 临时封 vs 永久封?

## 6. 推荐"先跑通"配置(默认值,可写代码)

为了不阻塞 W2~W4,先按下面默认值实现,后面策划再调:

```
# 大厅 PvP
死亡惩罚:A(轻惩罚,5s 复活,无掉落)
新人保护:A(等级 < 10 免战)
击杀奖励:固定 100 金币 + 50 经验
安全区:NPC 周围 30m 半径
阵营:无,全员可互打,同队不互伤

# 战斗
模式:5v5
投降:4/5 同意,15min 后可投
AFK:3 分钟无操作判负

# 段位(初版)
青铜(0~999)→ 白银(1000~1499)→ 黄金(1500~1999)
→ 铂金(2000~2499)→ 钻石(2500~2999)→ 大师(3000+)
赛季:暂无,持续累计

# MMR
Glicko-2,队伍 MMR = 队员加权(队长 1.0,队员 1.0,solo 1.0)

# 反作弊
移动速度阈值:maxSpeed × 1.2
异常 5 次/小时 → 自动 7 天封
```

**这套默认值能让 W2~W4 全部进度推进**。后期策划真要改属于"换数值",改 1 行 config 即可,不影响架构。
